## 1. 产品定位

| 项目 | 内容 |
|------|------|
| **产品名称** | TokenMachine |
| **产品定位** | 一站式 AI 模型部署、调度、管理平台 |
| **目标用户** | 企业 IT 部门、AI 创业公司、研究机构 |
| **核心价值** | 降本（私有化部署）、灵活（多硬件多模型）、可控（数据自主）、易用（一键部署） |

**核心价值**:
- **降本**: 私有化部署，一次投入长期使用
- **灵活**: 支持多种硬件（NVIDIA、昇腾、沐曦、海光）、多种模型
- **可控**: 数据不出域，完全自主可控
- **易用**: 一键部署，开箱即用

## 2. 核心功能

### 2.1 模型部署管理

#### 2.1.1 模型仓库

| 功能 | 描述 |
|------|------|
| **本地模型管理** | 支持上传本地模型文件 |
| **模型下载** | 支持 HuggingFace、ModelScope（可断点续传） |
| **模型版本** | Git-like 版本管理（v1.0, v2.0...） |
| **量化变体** | FP16、INT8、FP4、FP8 等量化格式 |
| **模型指纹** | MD5/SHA256 校验，确保完整性 |

#### 2.1.2 部署配置

| 功能 | 描述 |
|------|------|
| **一键部署** | 选择模型 → 配置参数 → 一键启动 |
| **多环境部署** | 开发、测试、预发、生产环境隔离 |
| **副本管理** | 单机多副本，负载均衡 |
| **灰度发布** | 版本逐步切换，支持回滚 |
| **A/B 测试** | 多版本并行，流量权重控制 |
| **GPU 绑定** | 自动分配 GPU 或手动指定 |

#### 2.1.3 推理引擎

| 引擎 | 支持状态 | 适用场景 |
|------|----------|----------|
| **vLLM** | ✅ 默认 | 通用场景，生态成熟 |
| **SGLang** | 🔧 P1 | 高并发场景，吞吐量高 |
| **TensorRT-LLM** | 📋 P2 | 性能极致场景 |
| **Chitu** | 📋 P2 | 国产芯片（昇腾、沐曦、海光） |

#### 2.1.4 模型类型支持

| 模型类型 | 支持状态 | 应用场景 |
|----------|----------|----------|
| **LLM** | ✅ P0 | 对话、文本生成 |
| **Embedding** | ❌ | 暂不支持（未来 P2） |
| **Reranker** | ❌ | 暂不支持（未来 P2） |
| **图像生成** | ❌ | 不支持 |
| **语音** | ❌ | 不支持 |

### 2.2 集群与节点管理

#### 2.2.1 节点管理

| 功能 | 描述 |
|------|------|
| **添加节点** | 通过 Token 注册接入新 Worker 节点，类似 Kubernetes 节点加入机制 |
| **节点列表** | 查看所有已加入的 Worker 节点，显示节点状态、IP、GPU 信息 |
| **节点下线** | 优雅下线节点，自动迁移已有模型实例 |
| **节点标签** | 为节点打标签（如：gpu-type、zone、environment），用于调度选择 |
| **节点隔离** | 标记节点为不可调度，维护模式 |

#### 2.2.2 资源管理

| 功能 | 描述 |
|------|------|
| **GPU 发现** | 自动检测节点上的 NVIDIA、昇腾、沐曦、海光 GPU |
| **资源监控** | 实时显示 GPU 利用率、温度、显存、功率 |
| **健康检查** | 定期检测节点和 GPU 状态，故障自动告警 |
| **资源分配** | 按租户/团队隔离 GPU 资源池 |

#### 2.2.3 调度策略

| 功能 | 描述 |
|------|------|
| **自动调度** | 根据资源需求自动选择合适节点部署模型 |
| **手动调度** | 支持手动指定节点和 GPU |
| **副本分布** | 支持 Spread（高可用）和 Binpack（节省资源）策略 |
| **弹性扩缩容** | 基于队列深度自动增减模型副本数 |

#### 2.2.4 支持的硬件

| 硬件厂商 | 支持状态 | 推理引擎 |
|----------|----------|----------|
| **NVIDIA** | ✅ P0 | vLLM, SGLang, TensorRT-LLM |
| **华为昇腾** | 🔧 P2 | Chitu (MindIE) |
| **沐曦** | 🔧 P2 | Chitu |
| **海光** | 🔧 P2 | Chitu |

### 2.3 API 网关

#### 2.3.1 OpenAI 兼容 API

| API 端点 | 状态 | 描述 |
|---------|------|------|
| `POST /v1/chat/completions` | ✅ P0 | 聊天补全，支持流式输出 |
| `GET /v1/models` | ✅ P0 | 列出可用模型 |

#### 2.3.2 请求路由

| 功能 | 描述 |
|------|------|
| **语义路由** | 根据请求内容（模型名称、参数）智能路由到对应后端 |
| **API 聚合** | 多个模型实例聚合为单一 API 端点，对外统一接口 |
| **权重分配** | 为不同模型实例分配流量权重（如：A 版本 80%，B 版本 20%） |
| **动态调度** | 基于后端负载（队列深度、响应时间）动态分配请求 |
| **故障转移** | 后端实例故障时自动切换到健康实例 |

#### 2.3.3 认证鉴权

| 功能 | 描述 |
|------|------|
| **API Key 管理** | 创建、撤销、查看 API Key，支持前缀显示 |
| **API Key 权限** | 为 API Key 分配可访问的模型和操作权限 |
| **配额管理** | 按 API Key 限制 Token 消耗、请求数、并发数 |
| **使用统计** | 记录每个 API Key 的调用统计和费用 |

#### 2.3.4 流式输出

| 功能 | 描述 |
|------|------|
| **SSE** | Server-Sent Events，实时流式输出 |
| **请求取消** | 支持中断正在生成的请求 |

### 2.4 测试场 (Playground)

#### 2.4.1 对话测试

#### 2.4.2 MCP 工具调用测试

#### 2.4.3 Agent 框架测试

| 功能 | 描述 |

| **Agent 模板** | 内置 Agent 模板（ReAct、Plan-and-Solve、COT）|
| **知识库管理** | 上传文档（PDF、TXT、Markdown），构建向量知识库 |

#### 2.4.4 Benchmark 测试

| 功能 | 描述 |
|------|------|
| **精度测试** | • MMLU: 多任务语言理解基准<br>• GSM8K: 数学推理<br>• HumanEval: 代码生成<br>• TruthfulQA: 事实性问答<br>• BBH: 大规模密集任务<br>• C-Eval: 中文综合评估 |
| **性能测试** | • Time to First Token (TTFT): 首字延迟<br>• Tokens Per Second (TPS): 生成速度<br>• Request Per Second (RPS): 吞吐量<br>• P50/P95/P99 Latency: 延迟分位数<br>• 并发测试: 模拟多用户并发 |
| **测试集管理** | 上传自定义测试集，支持 JSONL/CSV 格式 |
| **测试配置** | 选择测试集、并发数、测试轮数、超时设置 |
| **实时报告** | 测试过程中实时显示进度和中间结果 |
| **结果对比** | 多模型/多配置测试结果横向对比图表 |
| **报告导出** | 导出测试报告（PDF、JSON、Markdown）|
| **历史记录** | 保存测试历史，支持回溯和趋势分析 |
| **评分卡** | 综合评分系统，自动生成模型评分卡 |

### 2.6 监控与可观测性

#### 2.6.1 指标采集

| 类型 | 指标 | 采集方式 |
|------|------|----------|
| **业务层** | API 调用量、Token 消耗、用户活跃度 | Prometheus |
| **服务层** | QPS、延迟、并发数、错误率 | Prometheus |
| **基础设施** | GPU 利用率、温度、显存、CPU/内存 | Prometheus |

#### 2.6.2 告警规则

| 指标 | 阈值 | 级别 | 动作 |
|------|------|------|------|
| GPU 利用率 | > 95% | 警告 | 记录日志 |
| GPU 温度 | > 85°C | 紧急 | 自动降频 |
| API 错误率 | > 5% | 严重 | 发送告警 |
| 响应时间 | > 5s | 警告 | 扩容触发 |

#### 2.6.3 日志管理

| 功能 | 描述 |
|------|------|
| **结构化日志** | Loguru 日志格式，包含时间戳、级别、上下文 |
| **日志流** | WebSocket 实时推送日志到前端 |
| **日志查询** | 按时间、级别、关键词过滤 |
| **日志归档** | 定期归档到对象存储（未来） |

## 3. 功能优先级

### 3.1 Phase 1: MVP（12 周）

| 模块 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| **节点管理** | 添加/下线节点、节点列表、Token 注册接入 | P0 | 类似 K8s 节点管理 |
| **GPU 管理** | GPU 资源发现、状态监控、资源分配 | P0 | 自动检测可用 GPU |
| **模型部署** | 模型下载、版本管理、部署状态管理 | P0 | 支持断点续传 |
| **API 网关** | OpenAI Chat API、Models API、语义路由、权重分配 | P0 | 完全兼容 OpenAI |
| **API Key** | API Key 认证、创建、撤销、配额管理、权限控制 | P0 | Bearer Token |
| **监控** | Prometheus 指标、结构化日志 | P0 | GPU/模型/API 指标 |
| **Web UI** | 模型管理、集群管理、API Key 管理、仪表盘 | P0 | 部署/停止模型 |

### 3.2 Phase 2: 增强（3 个月）

| 模块 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| **计费系统** | Token 计费、配额管理、账单生成 | P1 | 按使用量计费 |
| **多租户** | 租户管理、用户管理、RBAC、资源隔离 | P1 | 企业级支持 |
| **动态调度** | 基于后端负载的智能调度 | P1 | 提升资源利用率 |
| **SGLang 后端** | 高并发推理优化 | P1 | 吞吐量提升 50% |
| **模型微调** | LoRA 训练、A/B 测试 | P1 | 模型优化 |

### 3.3 Phase 3: 企业版（6 个月）

| 模块 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| **Embedding 模型** | 文本嵌入 API | P2 | 向量检索、RAG |
| **Reranker 模型** | 重排序 API | P2 | 搜索结果重排序 |
| **国产芯片** | 昇腾、沐曦、海光支持 | P2 | Chitu 集成 |
| **异构计算** | CPU+GPU 混合推理 | P2 | KTransformers 集成 |
| **高级 RBAC** | 细粒度权限控制 | P2 | 审计日志 |
| **合规功能** | 等保三级、数据加密 | P2 | 企业合规 |
