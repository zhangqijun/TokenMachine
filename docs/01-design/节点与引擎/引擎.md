# 引擎管理设计文档

## 1. 概述

引擎管理是 TokenMachine 平台的核心功能之一，负责管理 AI 推理引擎的生命周期，包括安装、删除、版本管理和状态追踪。

### 1.1 功能定位

引擎管理的本质是 **私有镜像仓库的 Docker 镜像元数据管理**：
- 镜像存储在私有镜像仓库（如 Harbor）中
- 记录引擎版本信息、镜像地址、仓库路径等元数据
- 支持多种推理引擎：vLLM、MindIE、llama.cpp

### 1.2 核心功能

| 功能 | 描述 |
|------|------|
| 引擎列表 | 查看已安装和可用的引擎版本 |
| 引擎安装 | 从私有仓库拉取指定版本的推理引擎 Docker 镜像 |
| 引擎删除 | 从本地移除不再需要的引擎镜像 |
| 版本管理 | 同一引擎支持多版本并存 |
| 状态追踪 | 实时监控引擎安装状态和活跃部署数 |

## 2. 数据模型

### 2.1 BackendEngine

```python
class BackendEngine(Base):
    id: int                          # 主键
    engine_type: BackendEngineType   # 引擎类型 (VLLM, MINDIE, LLAMA_CPP)
    version: str                     # 版本号 (如 0.5.0)
    status: BackendEngineStatus      # 安装状态
    registry_url: str                # 私有镜像仓库地址
    image_name: str                  # 完整镜像名 (registry/image:tag)
    size_mb: int                     # 镜像大小 (MB)
    active_deployments: int          # 活跃部署数
    config: Dict                     # 运行时配置
    env_vars: Dict                   # 环境变量
    installed_at: datetime           # 安装时间
    created_at: datetime             # 创建时间
```

### 2.2 枚举类型

**BackendEngineType**
- `VLLM`: 高性能推理引擎，支持 PagedAttention
- `MINDIE`: 华为昇腾推理引擎，国产化适配
- `LLAMA_CPP`: 轻量级引擎，支持 CPU 和 Apple Silicon

**BackendEngineStatus**
- `NOT_INSTALLED`: 未安装
- `INSTALLED`: 已安装

## 3. 服务层设计

### 3.1 BackendEngineService

```python
class BackendEngineService:
    @staticmethod
    def list_engines(
        db: Session,
        engine_type: Optional[BackendEngineType] = None
    ) -> List[Dict[str, Any]]:
        """列出引擎，支持按类型过滤"""

    @staticmethod
    def install_engine(
        db: Session,
        engine_type: BackendEngineType,
        request: BackendEngineInstallRequest
    ) -> BackendEngine:
        """从私有仓库安装引擎"""

    @staticmethod
    def delete_engine(
        db: Session,
        engine_type: BackendEngineType,
        version: str
    ) -> bool:
        """删除引擎"""

    @staticmethod
    def get_engine_stats(
        db: Session,
        engine_type: BackendEngineType,
        version: str
    ) -> Dict[str, Any]:
        """获取引擎统计信息"""
```

## 4. API 设计

### 4.1 REST API 端点

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /api/v1/backends | 获取引擎列表 |
| POST | /api/v1/backends/{engine_type}/install | 安装引擎 |
| DELETE | /api/v1/backends/{engine_type}/{version} | 删除引擎 |
| GET | /api/v1/backends/{engine_type}/{version}/stats | 获取引擎统计 |

## 5. 镜像管理流程

### 5.1 安装流程

```
1. 验证私有仓库连接
   └─> docker login ${registry_url}

2. 拉取镜像
   └─> docker pull ${registry_url}/${image_name}

3. 如需要，推送到本地仓库
   └─> docker tag ${image_name} ${local_registry}/${image_name}
   └─> docker push ${local_registry}/${image_name}

4. 记录元数据到数据库
   └─> BackendEngine.status = INSTALLED
```

### 5.2 删除流程

```
1. 检查是否有活跃部署
   └─> active_deployments == 0

2. 删除本地镜像
   └─> docker rmi ${image_name}

3. 删除数据库记录
   └─> db.delete(engine)
```

## 6. 私有镜像仓库配置

### 6.1 仓库配置

| 配置项 | 说明 | 示例 |
|--------|------|------|
| registry_url | 私有仓库地址 | registry.tokenmachine.ai |
| registry_username | 仓库用户名 | admin |
| registry_password | 仓库密码 | (加密存储) |
| default_namespace | 默认命名空间 | vllm, mindie, llamacpp |

### 6.2 镜像命名规范

```
${registry_url}/${namespace}/${name}:${version}

示例：
registry.tokenmachine.ai/vllm/vllm-openai:v0.5.0
registry.tokenmachine.ai/mindie/mindie-serving:v1.0.0
registry.tokenmachine.ai/llamacpp/llama-cpp-python:v0.2.0
```

## 7. 边缘情况处理

| 场景 | 处理方式 |
|------|----------|
| 安装重复版本 | 抛错：`{engine} {version} is already installed` |
| 删除有活跃部署的引擎 | 抛错：`Cannot delete engine with {n} active deployments` |
| 删除不存在的引擎 | 抛错：`{engine} {version} not found` |
| 私有仓库认证失败 | 抛错：认证失败，请检查仓库凭据 |
| 镜像拉取失败 | 抛错：拉取失败，请检查网络和镜像地址 |
