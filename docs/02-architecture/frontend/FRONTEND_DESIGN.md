## 1. 设计概述

### 1.1 设计理念

TokenMachine 前端设计遵循以下核心原则：

| 原则 | 说明 | 应用场景 |
|------|------|----------|
| **专业性** | 企业级界面标准，严谨的视觉层次 | 数据展示、表单设计 |
| **效率优先** | 减少操作步骤，快速完成任务 | 模型部署、配置调整 |
| **实时反馈** | 数据实时更新，状态即时可见 | GPU 监控、部署状态 |
| **容错性** | 明确的错误提示，预防误操作 | 删除确认、表单验证 |
| **一致性** | 统一的交互模式和视觉语言 | 全局导航、操作按钮 |

### 1.2 用户角色

| 角色 | 描述 | 核心诉求 |
|------|------|----------|
| **系统管理员** | 负责平台运维和资源管理 | 快速定位问题、资源状态一目了然 |
| **算法工程师** | 负责模型部署和测试 | 简化部署流程、便捷的调试工具 |
| **开发者** | 使用 API 进行集成 | 清晰的 API 文档、便捷的 Key 管理 |

## 2. 页面架构设计

### 2.1 整体布局

采用经典的 **侧边栏 + 顶部栏 + 内容区** 布局：

```
┌─────────────────────────────────────────────────────────┐
│  顶部导航栏 (56px)                                        │
│  Logo | 主导航 |             用户信息 通知          │
├──────┬──────────────────────────────────────────────────┤
│      │                                                  │
│ 侧  │                                                  │
│ 边  │          内容区域                                 │
│ 栏  │          (自适应高度)                             │
│      │                                                  │
│(240)│                                                  │
│      │                                                  │
│      │                                                  │
└──────┴──────────────────────────────────────────────────┘
```

**布局特点**：
- **固定高度导航栏**：始终可见，提供全局导航
- **可折叠侧边栏**：支持收起/展开，最大化内容区域
- **全屏内容区**：使用剩余所有空间，支持内部滚动
- **面包屑导航**：二级页面显示路径，方便返回

### 2.2 导航结构

#### 2.2.1 顶部导航栏

**位置**：固定在顶部，高度 56px

**组成元素**：
| 区域 | 内容 | 交互 |
|------|------|------|
| **左侧** | Logo、产品名称 | 点击回到首页 |
| **中间** | 主导航菜单 | 高亮当前页面 |
| **右侧** | 搜索框、帮助文档、通知图标、用户头像 | 下拉菜单/跳转 |

#### 2.2.2 侧边栏

**位置**：左侧固定，宽度 240px（可折叠至 64px）

**菜单结构**：
```
📊 仪表盘
🤖 模型管理
🖥️  集群管理
🔑 API 密钥
🧪 测试场
⚙️  系统设置
```

**折叠状态**：
- 展开时：显示图标 + 文字
- 折叠时：仅显示图标，悬停显示文字提示

- 指标卡片：实时轮询（5秒间隔）
- 图表数据：每分钟更新一次
- 使用乐观更新 + 轮询补偿策略

- 进度条显示（0-100%）

## 5. 数据流与状态管理

### 5.1 状态管理架构

采用 **Zustand** 进行状态管理，分层设计：

```
全局状态 (Store)
├── userStore (用户状态)
│   ├── userInfo
│   ├── permissions
│   └── isAuthenticated
│
├── modelStore (模型状态)
│   ├── models: Model[]
│   ├── selectedModel: Model
│   ├── filters: FilterState
│   └── loading: boolean
│
├── deploymentStore (部署状态)
│   ├── deployments: Deployment[]
│   ├── selectedDeployment: Deployment
│   ├── instances: Instance[]
│   └── statusMap: Map<string, Status>
│
├── clusterStore (集群状态)
│   ├── workers: Worker[]
│   ├── gpus: GPU[]
│   ├── metrics: Metrics
│   └── loading: boolean
│
├── apiStore (API 密钥状态)
│   ├── apiKeys: ApiKey[]
│   ├── usageStats: UsageStats
│   └── loading: boolean
│
└── uiStore (UI 状态)
    ├── sidebarCollapsed: boolean
    ├── notifications: Notification[]
    ├── theme: 'light' | 'dark'
    └── currentRoute: string
```

### 5.2 数据流向

**单向数据流**：
```
用户操作 → Action → API 请求 → 更新 Store → 重新渲染 → UI 更新
```

**示例：创建部署**
```
1. 用户点击"创建部署"
2. 调用 deploymentStore.createDeployment(data)
3. 发送 POST /api/v1/deployments
4. 乐观更新：deploymentStore.deployments.push(newDeployment)
5. 等待响应
6. 成功：更新 deploymentStore 状态
7. 失败：回滚乐观更新，显示错误提示
```

### 5.3 API 集成

**API 客户端封装**：
- 统一的错误处理
- 自动重试机制
- 请求/响应拦截器
- Token 自动刷新

**API 分类**：

| 类别 | 前缀 | 说明 |
|------|------|------|
| **认证** | `/api/v1/auth` | 登录、登出、Token 刷新 |
| **模型管理** | `/api/v1/models` | 模型 CRUD、下载 |
| **部署管理** | `/api/v1/deployments` | 部署 CRUD、启停 |
| **集群管理** | `/api/v1/cluster` | Worker 管理 |
| **API 密钥** | `/api/v1/api-keys` | 密钥 CRUD |
| **OpenAI API** | `/v1` | Chat Completions、Models |
| **监控** | `/api/v1/metrics` | Prometheus 指标查询 |

### 11.3 实时数据更新机制

#### 11.3.1 Watch API + Web Worker

**架构**：
```
┌─────────────┐     SSE      ┌─────────────┐     Worker     ┌─────────────┐
│   前端组件   │ ←────────→  │  Hook 层    │ ←──────────→  │  JSON 解析   │
│  (UI 渲染)  │   实时流     │ useChunkReq │   流式解析     │  (后台线程)  │
└─────────────┘             └─────────────┘               └─────────────┘
```

**核心 Hook**：
```typescript
// useChunkRequest.ts
const useChunkRequest = () => {
  const [data, setData] = useState([]);
  const workerRef = useRef<Worker>();

  const setupChunkRequest = async (url: string) => {
    // 1. 创建 Web Worker
    workerRef.current = new Worker('./json-parser-worker.ts');

    // 2. 监听 Worker 消息
    workerRef.current.onmessage = (e) => {
      const validJSON = e.data;
      updateDataList(validJSON);  // 乐观更新
    };

    // 3. 发起 SSE 请求
    await request(url, {
      params: { watch: true },
      onDownloadProgress: (e) => {
        const newData = sliceResponseData(e.response);
        workerRef.current.postMessage(newData);
      }
    });
  };

  return { setupChunkRequest };
};
```

**适用场景**：
- GPU 指标实时监控
- 模型部署状态更新
- Worker 心跳检测
- 日志流式输出

#### 11.3.2 数据更新策略

| 策略 | 说明 | 使用场景 |
|------|------|----------|
| **乐观更新** | 先更新 UI，后确认 | 用户操作反馈 |
| **轮询补偿** | SSE + 定时轮询 | 网络不稳定时 |
| **增量更新** | 仅更新变化项 | 大列表优化 |
| **去重合并** | 相同事件合并 | 防止闪烁 |

#### 11.5.1 useTableFetch 示例

const useTableFetch = <T,>({
  fetchAPI,
  deleteAPI,
  defaultQueryParams,
  watch = false
}: Options) => {
  const [dataSource, setDataSource] = useState<{
    dataList: T[];
    loading: boolean;
    total: number;
  }>({
    dataList: [],
    loading: false,
    total: 0
  });

  const [queryParams, setQueryParams] = useState(defaultQueryParams);

  const fetchData = async (params?: Partial<QueryParams>) => {
    const res = await fetchAPI({ ...queryParams, ...params });
    setDataSource({
      dataList: res.items,
      loading: false,
      total: res.pagination.total
    });
  };

  // 如果启用 watch，建立 SSE 连接
  useEffect(() => {
    if (watch) {
      const { setupChunkRequest } = useChunkRequest();
      setupChunkRequest(`${API_URL}?watch=true`);
    }
  }, []);

  return {
    dataSource,
    queryParams,
    fetchData,
    handlePageChange,
    handleSearch,
    handleDelete
  };
};
