# 模型部署功能前端设计文档

## 一、概述

本文档描述 TokenMachine 系统中模型管理页面的部署功能设计。该功能允许用户在模型管理界面中直接部署、配置、监控和管理模型实例。

### 设计原则

1. **不创建新页面**：在现有 Models.tsx 页面基础上增强功能
2. **模态框交互**：所有配置和操作通过 Modal 弹窗完成
3. **状态驱动**：模型卡片根据部署状态显示不同的操作按钮
4. **实时反馈**：兼容性检查和资源使用情况实时更新

#### 2.2.1 搜索框设计

```

#### 2.2.2 搜索功能说明

- **实时搜索**：输入后自动触发搜索（防抖 500ms）
- **支持搜索内容**：
  - 模型名称（如：llama-3-8b）
  - 模型 ID
  - 标签（如：对话、推理）
  - 创建者/厂商（如：Meta AI）
- **快捷键**：按 `/` 键快速聚焦搜索框
- **清空搜索**：点击"清空"按钮或按 ESC 键清空搜索条件
- **搜索历史**：记录最近 5 次搜索，点击搜索框显示下拉列表

#### 2.4.1 排序下拉菜单

```

└─────────────────────────────────────────────────────────────────────────┘

#### 2.4.2 排序选项

点击排序下拉菜单显示所有排序选项：

```
┌─────────────────────────────────┐
│  ○ 默认排序                     │
│  ● 名称 (A → Z)         [↑↓]   │
│  ○ 名称 (Z → A)                  │
│  ● 大小 (从小到大)       [↑↓]   │
│  ○ 大小 (从大到小)               │
│  ● 部署状态                       │
│  ● 发布日期 (最新优先)   [↑↓]   │
│  ● QPS (从高到低)         [↑↓]   │
│                                  │
│  [重置排序]                     │
└─────────────────────────────────┘
```

#### 2.4.3 排序字段说明

| 排序字段 | 升序/降序 | 说明 |
|---------|----------|------|
| 默认排序 | - | 按系统推荐顺序（考虑部署状态、使用频率） |
| 名称 | A→Z / Z→A | 按模型名称字母顺序 |
| 大小 | 从小到大 / 从大到小 | 按模型参数规模 |
| 部署状态 | - | 运行中优先，其次未部署，最后错误 |
| 发布日期 | 最新优先 / 最早优先 | 按模型导入时间 |
| QPS | 从高到低 / 从低到高 | 按当前 QPS（仅运行中模型） |

#### 2.4.4 排序交互

- **切换排序**：点击排序选项立即应用
- **升序/降序**：点击箭头图标切换排序方向
- **排序图标**：当前排序字段显示 ↑↓ 图标
- **重置排序**：恢复到默认排序

## 六、兼容性检查

### 6.1 检查项目

兼容性检查包括以下项目：

1. **GPU 内存检查**
   - 计算模型所需显存
   - 检查可用显存是否充足
   - 返回：✅充足 / ⚠️ 紧张 / ❌不足

2. **后端支持检查**
   - 检查选择的后端是否支持该模型
   - 检查后端版本是否兼容
   - 返回推荐后端和版本

3. **Worker 可用性检查**
   - 检查选中的 Worker 是否在线
   - 检查 Worker 标签是否匹配
   - 返回可用 Worker 列表

4. **GPU 可用性检查**
   - 检查选中的 GPU 是否可用
   - 检查 GPU 是否有足够的显存
   - 返回可用 GPU 列表

5. **配置合理性检查**
   - 检查 Tensor Parallel 设置是否合理
   - 检查量化方式是否支持
   - 返回优化建议

### 6.2 检查结果展示

兼容性检查结果以三种状态展示：

- ✅ **通过**：配置可用，可以部署
- ⚠️ **警告**：配置可用，但有优化建议
- ❌ **错误**：配置不可用，需要修改

### 14.9 筛选逻辑实现

```typescript
// 筛选逻辑示例
const applyFilters = (
  models: DeployedModel[],
  filters: FilterState,
  keyword: string
): DeployedModel[] => {
  let result = [...models];

  // 1. 关键词搜索
  if (keyword.trim()) {
    const lowerKeyword = keyword.toLowerCase();
    result = result.filter(model => {
      return (
        model.name.toLowerCase().includes(lowerKeyword) ||
        model.id.toLowerCase().includes(lowerKeyword) ||
        model.tags.some(tag => tag.toLowerCase().includes(lowerKeyword)) ||
        model.creator.toLowerCase().includes(lowerKeyword)
      );
    });
  }

  // 2. 模型类型筛选
  if (filters.modelTypes.length > 0) {
    result = result.filter(model =>
      filters.modelTypes.includes(model.modelType)
    );
  }

  // 3. 标签筛选（多选：OR 关系）
  if (filters.tags.length > 0) {
    result = result.filter(model =>
      filters.tags.some(tag => model.tags.includes(tag))
    );
  }

  // 4. 系列/厂商筛选
  if (filters.series.length > 0) {
    result = result.filter(model =>
      filters.series.includes(model.series)
    );
  }

  // 5. 上下文筛选
  if (filters.contextLength.length > 0) {
    result = result.filter(model => {
      const contextMap: Record<string, number> = {
        '8k': 8192,
        '16k': 16384,
        '32k': 32768,
        '64k': 65536,
        '128k': 131072
      };
      return filters.contextLength.some(size => {
        const modelContext = contextMap[model.context?.toLowerCase() || ''];
        return modelContext >= (contextMap[size] || 0);
      });
    });
  }

  // 6. 规格筛选
  if (filters.sizeCategory.length > 0) {
    result = result.filter(model => {
      const size = model.size; // 单位：B（参数量）
      if (filters.sizeCategory.includes('small') && size < 10_000_000_000) return true;
      if (filters.sizeCategory.includes('medium') && size >= 10_000_000_000 && size < 50_000_000_000) return true;
      if (filters.sizeCategory.includes('large') && size >= 50_000_000_000) return true;
      return false;
    });
  }

  // 7. 发布日期筛选
  if (filters.dateRange && filters.dateRange !== 'all') {
    const now = Date.now();
    const dayMap: Record<string, number> = {
      '7days': 7 * 24 * 60 * 60 * 1000,
      '30days': 30 * 24 * 60 * 60 * 1000,
      '90days': 90 * 24 * 60 * 60 * 1000
    };
    const cutoff = now - (dayMap[filters.dateRange] || 0);
    result = result.filter(model => {
      const createdAt = new Date(model.createdAt || '').getTime();
      return createdAt >= cutoff;
    });
  }

  // 8. 部署状态筛选
  if (filters.deploymentStatus.length > 0) {
    result = result.filter(model =>
      filters.deploymentStatus.includes(model.deploymentStatus)
    );
  }

  return result;
};
```

### 14.10 排序逻辑实现

```typescript
// 排序逻辑示例
const applySort = (
  models: DeployedModel[],
  sortField: SortField,
  sortOrder: SortOrder
): DeployedModel[] => {
  const sorted = [...models];

  sorted.sort((a, b) => {
    let comparison = 0;

    switch (sortField) {
      case 'name':
        comparison = a.name.localeCompare(b.name);
        break;

      case 'size':
        comparison = a.size - b.size;
        break;

      case 'status':
        // 部署状态优先级：running > not_deployed > error
        const statusPriority = { 'running': 3, 'not_deployed': 2, 'error': 1 };
        comparison = statusPriority[a.deploymentStatus] - statusPriority[b.deploymentStatus];
        break;

      case 'date':
        comparison = new Date(b.createdAt || '').getTime() - new Date(a.createdAt || '').getTime();
        break;

      case 'qps':
        // 运行中的模型排在前面
        if (a.deploymentStatus === 'running' && b.deploymentStatus !== 'running') return -1;
        if (a.deploymentStatus !== 'running' && b.deploymentStatus === 'running') return 1;
        comparison = (a.metrics?.qps || 0) - (b.metrics?.qps || 0);
        break;

      default:
        // 默认排序：综合排序
        // 优先级：运行中 > 最近部署 > 使用频率 > 名称
        if (a.deploymentStatus !== b.deploymentStatus) {
          const statusPriority = { 'running': 2, 'not_deployed': 1, 'error': 0 };
          return statusPriority[b.deploymentStatus] - statusPriority[a.deploymentStatus];
        }
        comparison = a.name.localeCompare(b.name);
    }

    return sortOrder === 'asc' ? comparison : -comparison;
  });

  return sorted;
};
```
